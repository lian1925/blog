body 元素 6 | HTML

作者： Lian

日期： 2018年04月24日 17:04

body元素

body元素是网页的主体内容。大部分的网页的主要工作是编辑文本内容，以便浏览器能正确显示。
大部分的文本结构由标题和段落组成。 不管是小说、报刊、教科书还是杂志等。

标题(heading)元素
用于描述标题元素，包含六个不同的级别，h1到h6，其中h1级别最高，h6级别最低。

p元素
表示文本的一个段落。该元素是块级元素，通常表现为一整块与相邻文本分离的文本。以前的程序员喜欢省略p元素的结束标签，虽然浏览器不会报错，但这是不好的习惯，这里特别提醒。

div元素
这是开发中使用最多的元素，在语义上不代表任何特定类型的内容。它一般结合class和id属性，对相关元素进行分组，以便更好进行样式处理。

span元素
常用的行内元素，一般与p元素嵌套使用。

em元素
用于需要加粗的文本内容，表示强调。

strong元素
表示文本十分重要，一般用粗体。与em同时表示强调，但strong强调意味更加重。

pre元素
表示预定义格式文本，这些文本通常按照原文件中的编排，以等宽字体的形式展现出来。
比如：文本块中有需要行或者垂直对齐的文本,你希望这些内容即使在浏览器窗口大小改变时也不会发生变化，这种情况可以采用pre标签。
示例：

<pre><span class="line"><span class="tag">&lt;<span class="name">div</span> <span class="attr">id</span>=<span class="string">"app"</span>&gt;</span></span><br><span class="line">  {{ message }}</span><br><span class="line"><span class="tag">&lt;/<span class="name">div</span>&gt;</span></span><br></pre>

ul元素
ul，英文全称为unordered list，无序列表，表示没有特定顺序的一种列表(如洗衣店列表或者杂货店列表)。它们用 和 标签来定义边界，列表数据项item用

  • 标签来进行标记。
    示例

    • <ul>
  <li>first item</li>
  <li>second item</li>
  <li>third item</li>
</ul>

    ol元素
    ol,英文全称ordered list，有序列表，表示带数字编号的列表。有序列表用

      标签来限定边界，格式与无序列表相同。
      常用属性：
      1、reversed
      reverse，中文是翻转，默认情况，列表是顺序排序。不过，如果你需要逆序，可通过配置reversed属性来实现。它是布尔属性。
      2、start
      整数属性，规定列表序号的开始值。
      3、type
      规定序号类型。
      ‘1’，表示数字编号（默认）
      ‘a’，表示小写字母编号
      ‘A’，表示大写字母编号
      ‘i’，表示小写罗马数字编号
      ‘I’，表示大写罗马数字编号
      示例：

    <ol start="7">
  <li>first item</li>
  <li>second item</li>
  <li>third item</li>
</ol>

    form元素
    表示了文档中的一个区域，这个区域包含有交互控制元件，如文本输入框，单选按键，多选按键，下拉菜单等，用于获取用户的输入数据。
    常见属性：
    1、action
    规定接收和处理表单数据的应用程序的URL。该值有可能被button和input的formation属性覆盖。

    2、enctype
    浏览器在把数据传给服务器之前，会专门对表单的数据进行编码，这样在传输过程中数据才不会被打乱或破坏。
    默认格式是application/x-www-form-urlencoded。
    当传送的表单数据包括文件时，一般采用多部分表单格式 multipart/form-data。
    当无法访问表单处理服务器，需要将表单数据通过邮件发送时，才会采用text/plain格式。
    该值有可能会被button和input的formenctype属性覆盖。

    3、accept-charset
    指定字符集，服务器必须支持该字符集，才能正确解释表单中的数据。
    可不填，默认值为unknown，意味着表单的字符集与该HTML的字符集相同。

    4、method
    必填，有两种方法：post get
    如果采用Post方法，浏览器将会安装下面两步来发送数据。首先，浏览器与服务器建立联系，一旦建立联系成功，浏览器按分段传输的方法将数据发送给服务器。
    如果采用get方法，首先，浏览器与服务器建立联系，一旦建立联系，浏览器会将所有数据附在表单的url之后，用问号隔开，一次性发送给服务器。

    5、target
    用来指示在提交表单之后，在哪里显示收到的回复。
    _self: 在当前HTML4或HTML5文档页面重新加载返回值。这个是默认值。
    _blank: 以新的HTML4或HTML5文档窗口加载返回值。

    <!-- 一个简单的表单，这个表单会发送一个 GET 请求 -->
<form action="">
  <label for="GET-name">Name:</label>
  <input id="GET-name" type="text" name="name">
  <input type="submit" value="Save">
</form>

    fieldset 元素
    通常用来对表单中的控制元素进行分组(也包括 label 元素)
    示例：

    <form action="test.php" method="post">
  <fieldset>
    <legend>Title</legend>
    <input type="radio" id="radio"> <label for="radio">Click me</label>
  </fieldset>
</form>

    input
    常用的表单交互控件，用于接收用户的输入数据。可以实现文本输入，多选列表，提交按键等功能。没有结束标签。
    常见属性如下：
    1、type,必填，表明控件的类型
    2、text，文本输入框，默认值。
    3、checkbox，复选框，必须使用value属性定义此控件被提交的值。
    4、file，可以让用户选择文件。一般结合accept属性，定义文件类型。
    5、password，值被隐藏的文本输入框
    6、radio，单选按键，必须使用value属性定义此按键被提交的值。
    7、submit，表单提交的按键。

    checked
    如果该元素的type属性的值为radio或者checkbox,则该布尔属性的存在与否表明了该控件是否是默认选择状态.

    name
    控件名称，与表单数据一起提交

    required
    指定该元素为必填项。
    示例：

    <!-- 一个常见的表单,包含了一些input元素 -->
<form action="getform.php" method="get">
    First name: <input type="text" name="first_name" /><br />
     Last name: <input type="text" name="last_name" /><br />
        E-mail: <input type="email" name="user_email" /><br />
<input type="submit" value="Submit" />
</form>

    textarea
    用于创建一个多行文本的输入区域。
    rows和cols属性，分别定义矩形区域的横向与纵向维度。建议大家设置这些属性，因为浏览器通常有个习惯，就是为textarea设置最小的可读区域，且用户无法更改它的大小。

    <textarea name="textarea" rows="10" cols="50">Write something here</textarea>

    select
    用于构建下拉菜单，菜单内的选项为option元素。
    name
    控件名称

    示例：

    <!-- The second value will be selected initially -->
<select name="select">
  <option value="value1">Value 1</option>
  <option value="value2" selected>Value 2</option>
  <option value="value3">Value 3</option>
</select>
