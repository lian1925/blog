安全 1 | 前端卷

作者： Lian

日期： 2018年03月14日 19:03

1 什么是HTTPS?

答：

HTTPS，全称Hyper Text Transfer Protocol over Secure Socket Layer。

为了提升通信安全性，在http的基础上，增加了ssl/tls协议层。

http使用明文通信，主要有3方面风险：

窃听风险，第三方可以截获通信内容
篡改风险，第三方可以修改通信内容
冒充风险，第三方可以冒充他人身份参与通信

SSL/TLS协议，设计的初衷就是为了解决这三大风险：

加密信息，防止第三方窃听
增加校验机制，一旦信息篡改，通信双方立刻察觉
分配身份证书，防止身份被冒充

SSL/TLS协议的基本思路是 公钥加密：

客户端先向服务器端索要公钥，然后用公钥加密信息，服务器收到密文后，用自己的私钥解密。

2 这里有两个问题

1、如何保证公钥不被篡改？

解决方法：将公钥放在数字证书中。只要证书是可信的，公钥就是可信的。

2、公钥加密计算量太大，如何减少耗用的时间？

解决方法：每一次对话（session），客户端和服务器端都生成一个”对话密钥”（session key），用它来加密信息。由于”对话密钥”是对称加密，所以运算速度非常快，而服务器公钥只用于加密”对话密钥”本身，这样就减少了加密运算的消耗时间。

3 基本过程

SSL/TLS协议的基本过程是这样的：

1、客户端向服务器端索要并验证公钥。
2、双方协商生成”对话密钥”。
3、双方采用”对话密钥”进行加密通信。
