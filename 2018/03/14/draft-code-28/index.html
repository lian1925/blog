性能 | 面试

作者： Lian

日期： 2018年03月14日 18:03

1、如何提升前端的性能问题？

答：

前端的性能问题可能多种多样，并不存在既定的套路。解决的原则是，具体分析影响性能的关键要素，并设法解决。

要解决性能问题，我们首先得了解，从用户输入域名，到网站的内容加载完毕，这中间发生了哪些事情。

1、你在浏览器输入一个url

2、浏览器通过域名获取ip

这个过程是通过DNS完成的。
首先，查找浏览器的DNS缓存记录，若有返回ip。
若无，查找路由器的 DNS缓存记录，
若无，从ISP的DNS服务器，递归搜索，直到找到ip

3、浏览器通过ip，向web服务器发送HTTP请求，

4、web服务器，收到请求，判断参数，进行处理，并给出应答。

5、浏览器获得相应内容，并在页面正确显示。

浏览器在显示过程中，发现需要获取其他的标签内容，如图片，js, css，则继续发送请求，接收后正确显示。

第五步，称为页面的渲染过程。

大致过程分成5步为：

HTML代码转化成DOM
CSS代码转化成CSSOM（CSS Object Model）
结合DOM和CSSOM，生成一棵渲染树（包含每个节点的视觉信息）
生成布局（layout），即将所有渲染树的所有节点进行平面合成
将布局绘制（paint）在屏幕上

“生成布局”（flow）和”绘制”（paint）这两步，合称为”渲染”（render）。

构造dom树时，加载或执行js都会阻塞其进行；通过asyn和defer，可以解决阻塞问题。

构造渲染树时，加载css会阻塞其进行；

2、什么是重排和重绘？

答：

重新渲染，就需要重新生成布局和重新绘制。前者叫做”重排”（reflow），后者叫做”重绘”（repaint）。

重排和重绘，非常耗费资源，是导致网页性能低下的根本原因。

改善的几个技巧：

1、DOM 的多个读操作（或多个写操作），应该放在一起。不要两个读操作之间，加入一个写操作。

2、不要一条条地改变样式，而要通过改变class，或者csstext属性，一次性地改变样式。

3、使用 cloneNode() 方法，在克隆的节点上进行操作，然后再用克隆的节点替换原始节点。

3、如何提升首屏性能？

答：

提升网站性能，有个很重要的技巧，就是优化首页性能。
主要原则是，将可以延后加载的内容，尽量精简。
比如：图片按需加载；单页面应用，其他的不需要展示的页面，在打包阶段就需要隔离打包。

4、减少请求大小有哪些方法？

答：

压缩 js css html 文件；

图片压缩

js css 按需加载
