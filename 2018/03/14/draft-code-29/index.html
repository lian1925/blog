webpack | 性能

作者： Lian

日期： 2018年03月14日 19:03

webpack是当前最主流的前端模块打包工具。react 和 vue 均将其选为官方打包工具。

1、webpack 打包容易出现哪些性能问题？

答：

构建速度慢，即使很小改动，也需要长时间才能看到效果

基础库存在重复打包，复用率不高

随着项目复杂度增加，工程模块的体积随之增大

2、如何优化 webpack性能？

答：

从本身项目入手，检测代码结构，依赖使用是否合理

从webpack自身的优化手段出发，看看哪些api没有做好优化配置

从webpack自身的不足出发，做有针对性的扩展优化，进一步提升效率。

此处，webpack提供了一个好用的可视化资源分析工具:webpack-bundle-analyzer
自动帮你计算出各个模块在你的项目找那个的依赖和分布情况。

3、如何配置externals配置？

答：

在实际项目开发过程中，我们并不需要实时调试各种库的源码，这时可以考虑使用external选项了。

external就是把我们的依赖资源声明为一个外部依赖，然后通过script外链脚本引入，避免了重复打包的尴尬。

4、tree shaking

答:

我们的开发代码中，有时定义了一些代码，但在生产环境下并没有使用，这些代码块会增加打包容积，造成网页性能损耗。

tree shaking是webpack提供一种，帮助我们消除js代码中的无关代码的一种机制。

配置也很简单，安装uglifyjsplugin插件，并在webpack的配置文件中添加plugns的配置即可。

5、代码分割

答：

代码分割，code splitting，是webpack最重要的功能之一。

大致有三种方式：

一，通过入口entry，手动分割代码

二，通过 CommonsChunkPlugin，自动分割公用module

三，在代码中使用require.ensure函数，手动分割代码。
