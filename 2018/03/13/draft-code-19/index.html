1 对于新特性，怎么处理浏览器的兼容问题？

答：

现在大多数用户使用移动设备，这些浏览器大多支持较新的html5和css3特性，如果你们公司的客户刚好处于这种情况，则兼容则不是很严重的问题。

如果业务有要求解决，其思路有两：
一，为不兼容的特性写垫片，pollyfill，现有的html5和css3的特性，差不多都有人写了对应的plyfill，如果你不想自己写的话，可以下载使用。
比如：像FlashCanvas就是专门针对canvas元素的Polyfill

二， 最流行的检测库:Modernizr，Modernizr的功能就是检测当前浏览器对HTML5和CSS3各个特性的支持情况。

一种方案是，根据检测的情况，对旧版浏览器做优雅降级处理，比如 css3的text-shadow特性，可以制造漂亮的阴影效果，但是在旧版本ie不支持，则为之提供一个简单的替代样式方案。

另外，由于不同浏览器的默认样式不统一，为了给用户呈现一致的效果，可以采用normalize库，进行初始化。

2 如何解决屏幕适配

对应手机端和电脑端，怎么处理屏幕适配问题？

答：目前有两种解决思路。

一种是通过响应式布局来适配两种屏幕。它的特点是只有一套代码，全端公用。通过媒体查询屏幕宽度，来选用css样式，达到不同的适配效果。

这种方式，适用于相对简单的展示型网站，如企业官网，新闻网站，博客。

它的缺点是将pc于移动端的代码混在一起，这种开发方式其实违反了编程中非常重要的解耦原则。现在的移动端网站，用户提出了更高的挑战：更酷的用户效果，更快的加载速度，更灵敏的响应速度。因此移动端项目功能越来越多，复杂度越更高了。

因此，第二种方案就是，将pc和移动端项目分离，通过终端判断，向用户响应相应的项目。

3 安卓和苹果，如何达到一致的视觉效果？

答：

阿里巴巴的手淘团队，

虽然H5的页面与PC的Web页面相比简单了不少，但让我们头痛的事情是要想尽办法让页面能适配众多不同的终端设备。看看下图你就会知道，这是多么痛苦的一件事情：

而整个手淘设计师和前端开发的适配协作基本思路是：

选择一种尺寸作为设计和开发基准

定义一套适配规则，自动适配剩下的两种尺寸(其实不仅这两种，你懂的)

特殊适配效果给出设计效果

动态改写标签

给元素添加data-dpr属性，并且动态改写data-dpr的值

给元素添加font-size属性，并且动态改写font-size的值
