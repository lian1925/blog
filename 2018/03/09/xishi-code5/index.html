code 1

摘自

https://github.com/bailicangdu/vue2-elm/blob/master/src/style/mixin.scss

@mixin bis($url) { 
    background-image: url($url);
    background-repeat: no-repeat;
    background-size: 100% 100%;
}

理由：

如果需要复用某些scss的样式代码，使用@mixin是一种不错的用法。它可以像函数一样接收参数，且传参可以设定默认值。

code 2

摘自

https://github.com/bailicangdu/vue2-elm/blob/master/src/style/common.scss

@keyframes backOpacity{
   0%   { opacity: 1 }
   25%  { opacity: .5 }
   50%  { opacity: 1 }
   75%  { opacity: .5 }
   100% { opacity: 1 }
}

.animation_opactiy{
    animation: backOpacity 2s ease-in-out infinite;
}

理由：

这是使用@keyframes定义css动画的一种常见写法。

code 3

摘自

https://github.com/bailicangdu/vue2-elm/blob/master/src/style/common.scss

.clear:after{
    content: '';
    display: block;
    clear: both;
}

理由：

样式布局中，如果使用了浮动元素，为了不影响后面元素的位置，需要清除浮动。这里使用after是一种常规解法。

code 4

摘自

https://github.com/bailicangdu/vue2-elm/blob/master/src/store/mutations.js

[RECORD_SHOPDETAIL](state, detail) {
    state.shopDetail = detail;
}

理由：

该写法，用于对传参进行处理，与普通函数的用法相似。不一样的地方在于函数名在这里是以变量形式存在。

code 5

摘自

https://github.com/bailicangdu/vue2-elm/blob/master/src/store/mutations.js

//修改用户名
[RETSET_NAME](state,username) {
    state.userInfo = Object.assign({}, state.userInfo,{username})
}

理由：

在更新state对象时，推荐的做法是保证数据的immutable特性。使用Object.assign函数是一种很好的写法。
