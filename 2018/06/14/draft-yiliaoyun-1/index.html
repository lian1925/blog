前端选型

前端框架，是前端工程的根基，故选型问题需要慎之又慎。

当前，前端主流框架有两款：vue 和 react。为此，我联系团队实际，试图分析两者的优缺点。

Vue优点：
1、开发文档中文，受中国开发者欢迎。
2、据了解，我们团队中的6位前端(包括2位未报到)，之前都有Vue的项目经验。另外，听说平安内部很多前端项目都选用了Vue。

故，如果选用Vue框架，在前端工程师开发效率，共享代码，共享开发经验方面有很大的优势。

Vue缺点：
1、Vue框架对IE浏览器的支持度很差，最大限度可以支持IE 9 ，并且是需要添加很多补丁情况下。

2、如果需要支持 IE 8 ，Vue则完成无法胜用。且Vue的核心开发团队已明确表示放弃对旧版本浏览器的支持，故在可预期的未来，Vue对IE的支持都不乐观。

React优点：
1、对IE的支持相对较好，目前已有针对IE 8 兼容的解决方案。

2、由于我们项目的用户可能会涉及医院，政府事业单位，他们可能会使用旧版本的浏览器(这点需要和产品部门进行沟通确认)。
若是需要兼容旧版本浏览器，此时只能选用React。

React缺点：
1、开发资料主要是英文，需要开发者有好的英文功底。

2、开发复杂性相对Vue高点。

3、目前，平安内部大多项目使用Vue，若我们使用react，共享前端经验和填坑方面相对劣势。

4、目前市场上，招聘Vue程序员要比React程序好招。

express 前置开发

1、创建目录

mkdir api0

2、创建项目配置文件package.json

npm init

3、安装express 依赖

npm install express --save

4、创建新文件，app.js

var express = require('express');
var app = express();

app.get('/', function (req, res) {
  res.send('Hello World!');
});

var server = app.listen(3000, function () {
  var host = server.address().address;
  var port = server.address().port;

  console.log('Example app listening at http://%s:%s', host, port);
});

5、运行

node app.js

json web token

1、安装token库

npm install --save jsonwebtoken

2、生成token

var jwt = require('jsonwebtoken');

jwt.sign({
  exp: Math.floor(Date.now() / 1000) + (60 * 60),
  data: 'foobar'
}, 'secret');

3、验证token

// verify a token symmetric
jwt.verify(token, 'secret', function(err, decoded) {
  console.log(decoded.foo) // bar
});

请求参数：
{
username:’admin’,
password:’123’
}

返回：

{
