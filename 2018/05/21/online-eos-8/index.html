CLI Wallet【钱包】

作者： Lian

日期： 2018年05月21日 20:05

Overview【简介】

The program【编程】 keosd, located【位于】 in the eos/build/programs/keosd folder within the EOSIO/eos repository【仓库】, can be used to store【存储】 private keys that will be used to sign【签名】 transactions sent to the block chain【区块链】. keosd runs on your local machine and stores your private keys locally【本地】.

How to Run【运行】 keosd

Start【启动】 keosd on as follows:

$ keosd

By default, keosd creates【创建】 the folder ~/eosio-wallet and populates it with a basic【基本】 config.ini file. The location of the config file can be specified【配置】 on the command line using the –config-dir argument. The configuration file contains the http server endpoint【服务器终端】 for incoming http connections and other parameters for cross origin resource sharing【分享】.

By default, keosd stores wallets in the ~/eosio-wallet folder. Wallet files follow the naming convention【约定】 .wallet. For example, the default wallet will be stored【存储】 in a file named default.wallet. As other wallets are created【创建】, similar files will be created for each. For example, a wallet named “foo” will have a corresponding wallet file named foo.wallet. The location of the wallet data folder can be specified on the command line【命令行】 using the –data-dir argument.

Command Reference【参考】

The command line tool【命令行工具】 to interact【交互】 with keosd is called “cleos”. It is found in eos/build/programs/cleos folder【目录】.

It provides【提供】 the following commands to interact with keosd:

Create【创建】

$ cleos wallet create ${options}

Options【选项】:

-n,–name TEXT=default The name【名称】 of the new wallet

If you don’t provide an optional name it creates a default【默认】 wallet.

Open【打开】

Open an already【已经】 created wallet. You need to open a wallet to operate【操作】 on it.

$ cleos wallet open ${options}

Options【选择】:

-n,–name TEXT【文本】 The name of the wallet to open

Lock【上锁】

Locks【锁定】 a wallet.

$ cleos wallet lock ${options}

Options【选项】:

-n,–name TEXT【文本】 The name【名称】 of the wallet to lock

Note: Your wallets will be locked【加锁】 when your wallet manager (keosd or nodeos) is shut down【关闭】. You will need to unlock your wallets after restarting【重启】 the wallet manager【管理器】.

Unlock【解锁】 wallet

$ cleos wallet unlock ${options}

Options【选项】:

-n,–name TEXT【文本】 The name【名称】 of the wallet to unlock –password TEXT【文本】 The password【密码】 returned by wallet create

Import【导入】 private key into wallet

$ cleos wallet import ${options} key

Positionals:

key TEXT Private key in WIF format to import

Options【选项】:

-n,–name TEXT The name of the wallet to import key into

List【列表】

List opened【打开的】 wallets, * = unlocked

$ cleos wallet list

Keys【钥匙列表】

List of private keys from all unlocked wallets in WIF format.

$ cleos wallet keys

Managing【映射】 Wallets with nodeos

As an alternative【可选】 to using keosd to manage【管理】 your wallets, you can use nodeos to manage them. Using nodeos for wallet management is not the preferred【最好】 configuration, but is suitable【适用】 for development【开发】 and test【测试】. It is not recommended to simultaneously have keosd and nodeos doing wallet management; it won’t break anything, but you will likely find it very confusing【迷惑】!

To use nodeos for wallet management, you must configure nodeos to use the eosio::wallet_api_plugin. You can do this on the command line argument【命令行参数】 when starting nodeos, or by defining it in the nodeos config file. You will still use cleos as your interface【接口】 to perform wallet management tasks【任务】, the difference【区别】 being that cleos will direct【直接】 its requests to nodeos.

To start nodeos with the wallet plugin【插件】, include –plugin eosio::wallet_api_plugin on the nodeos command line.

To include the wallet using the nodeos config【配置】 file, add the following line to the nodeos config.ini file:

plugin = eosio::wallet_api_plugin

Note【注意】: If you use nodeos for wallet management, your wallets will be locked when nodeos is shut down. You will need to unlock your wallets after restarting nodeos.
