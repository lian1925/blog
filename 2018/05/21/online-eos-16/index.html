Comprehensive【理解】 Accounts【账户】 and Wallets【钱包】

作者： Lian

日期： 2018年05月21日 20:05

Comprehensive Accounts & Wallets Tutorial【教程】

Note This tutorial is geared towards use on a private【私有】 single node testnet【测试节点】, but will work on a public【公共】 testnet with minor modifications【改变】.

Tutorial Audience

This tutorial is for users who want to learn about wallet and account management【管理】, how to use cleos to manage wallets and accounts, and how the wallet and account management EOSIO components【组件】 interact with【交互】 each other. Additional information【信息】 can be found in the Command Reference【参考】.

What You’ll Learn【学习】

You’ll learn how to create and manage wallets, their keys【密钥】 and then use this wallet to interact with the blockchain【区块链】 through cleos. You will then learn how to create【创建】 accounts using cleos. The tutorial will introduce【介绍】 you to some of the interaction【交互】 between cleos, keosd, and nodeos to sign content published to the blockchain.

Prerequisites【条件】

Built and running copy【副本】 of cleos and keosd on your system.

Built and ready to run copy of nodeos on your system.

Basic understanding of command line interfaces【命令行接口】.

Note: Instructions【指令】 require slight modification【改变】 when applied【应用】 to a docker installation.

EOSIO Accounts【账户】 and Wallets【钱包】 Conceptual Overview

The diagram【框图】 below provides a simple conceptual view of accounts and wallets in EOSIO. While there are other supported deployment【部署】 configurations, this view matches the one that we will use for this tutorial.


Accounts and Wallets Conceptual Overview

The wallet can be thought of as a repository【仓库】 of public-private key pairs【密钥对】. These are needed to sign operations【操作】 performed on the blockchain. Wallets and their content are managed by keosd. Wallets are accessed using cleos.

An account can be thought of as an on-chain identifier【识别】 that has access permissions【权限】 associated with it (i.e., a security principal). nodeos manages the publishing of accounts and account-related actions on the blockchain【区块链】. The account management capabilities of nodeos are also accessed using cleos.

There is no inherent relationship【关系】 between accounts and wallets. Accounts do not know about wallets, and vice versa. Correspondingly, there is no inherent relationship between nodeos and keosd. Their basic【基本】 functions are fundamentally different【不同】. (Having said that, there are deployment configurations that blur the distinction. However, that topic【主题】 is beyond the scope of this tutorial.)

Where overlap occurs is when signatures【签名】 are required, e.g., to sign transactions【交易】. The wallet facilitates【帮助】 obtaining signatures in a secure【安全】 manner by storing【存储】 keys locally in an encrypted【加密】 store that can be locked. cleos effectively【有效】 serves as an intermediary between keosd key retrieval operations and nodeos account (and other) blockchain actions that require signatures【签名】 generated using those keys.

Creating【创建】 and Managing【管理】 Wallets

Open your terminal【终端】 and change【改变】 to the directory【目录】 where EOSIO was built. This will make it easier for us to interact with【交互】 cleos, which is a command line interface【命令行】 for interacting with nodeos and keosd.

$ cd /path_to_eos/build/programs/cleos

The first thing you’ll need to do is create a wallet; use the wallet create command of cleos

$ cleos wallet create
Creating wallet: default
Save password to use in the future to unlock this wallet.
Without password imported keys will not be retrievable.
"A MASTER PASSWORD"

A wallet called default【默认】 is now inside【内部】 keosd and has returned the master password【主密码】 for this wallet. Be sure【确保】 to save【保存】 this password somewhere safe. This password is used to unlock【解锁】 (decrypt) your wallet file.

The file【文件】 for this wallet is named default.wallet. By default, keosd stores wallets in the ~/eosio-wallet folder. The location【位置】 of the wallet data【数据】 folder can be specified on the command line【命令行】 using the –data-dir argument.

Managing【管理】 Multiple【多个】 Wallets and Wallet Names

cleos is capable【允许】 of managing multiple wallets. Each individual【个人】 wallet is protected【保护】 by different wallet master passwords. The example【示例】 below creates another wallet and demonstrates how to name it by using the -n argument【参数】.

$ cleos wallet create -n periwinkle
Creating wallet: periwinkle
Save password to use in the future to unlock this wallet.
Without password imported keys will not be retrievable.
"A MASTER PASSWORD"

Now confirm【确认】 that the wallet was created with your chosen【选择】 name.

$ cleos wallet list
Wallets:
[
  "default *",
  "periwinkle *"
]

It’s important to notice【注意】 the asterisk (*) after each listed【列出】 wallet, which means that the respective wallet is unlocked【解锁】. When using create wallet the resulting wallet is unlocked by default【默认】 for your convenience【方便】.

Lock that second【第二个】 wallet using wallet lock

$ cleos wallet lock -n periwinkle
Locked: 'periwinkle'

Upon running【运行】 wallet list again, you will see that the asterisk is gone, meaning that the wallet is now locked【加锁】.

$ cleos wallet list
Wallets:
[
  "default *",
  "periwinkle"
]

Unlocking【解锁】 a named wallet entails calling wallet unlock with an -n parameter【参数】 followed by the name of the wallet and then entering the wallet’s master password【主密码】 in the password prompt (yes, you can paste the password). Go ahead and grab the master key for the second wallet you created【创建】, execute【执行】 the command below and when the password prompt appears, paste and press enter. You’ll be presented with a confirmation【确认】.

$ cleos wallet unlock -n periwinkle

cleos will let you know【知道】 that the wallet was unlocked

Unlocked: 'periwinkle'

Note【注意】: You can also use【使用】 a –password argument followed【跟随】 by the master password to skip【跳过】 the prompt, but this results in your master password being visible【可见】 in the console history【历史】

Now check your progress【进度】:

$ cleos wallet list
Wallets:
[
  "default *",
  "periwinkle *"
]

The periwinkle wallet【钱包】 is followed by an asterisk, so it is now unlocked【解锁】.

Note: Interacting with【交互】 the ‘default’ wallet using the wallet command does not require【不需要】 the -n parameter

Restart【重启】 keosd now, and then go back to【返回】 where you were calling cleos and run the following command

$ cleos wallet list
Wallets:
[]

Wallets first【首先】 need to be opened before they can be operated【操作】 on, including listing them. The wallets were locked when you shut down【关闭】 keosd. When keosd was restarted, the wallet wasn’t open【打开】. Run【运行】 the following commands to open, then list the default wallet.

$ cleos wallet open
$ cleos wallet list
Wallets:
[
  "default"
]

Note: If you wanted to open a named【命名】 wallet, you would run $ cleos wallet open -n periwinkle.

You’ll notice【注意到】 in the last response that the default wallet is locked by default【默认】. Unlock the now; you’ll need it in the subsequent steps.

Run the wallet unlock command and paste【黏贴】 your default wallet’s master password when the password prompt appears【出现】.

$ cleos wallet unlock
Unlocked: 'default'

Check【检查】 that the wallet is unlocked:

$ cleos wallet list
Wallets:
[
  "default *"
]

The wallet is accompanied by an asterisk, so it’s unlocked.

You’ve learned how to create【创建】 multiple wallets, and interact with them in cleos. However, an empty wallet doesn’t do you much good. We will now learn to import【导入】 keys into the wallet.

Generating【生成】 and Importing【导入】 EOSIO Keys

There are several ways【方式】 to generate an EOSIO key pair【密钥对】, but this tutorial will focus on the create key command in cleos.

Generate【生成】 two public/private key pairs. Note the general【一般】 format of the keys.

$ cleos create key
Private key: XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
Public key: EOSXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
$ cleos create key
Private key: XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
Public key: EOSXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX

You now have two【两个】 EOSIO keypairs. At this point, these are just arbitrary【任意】 keypairs and by themselves have no authority【许可】.

If you followed all of the previous【之前】 steps, your default【默认】 wallet should be open and unlocked.

In this next step【步骤】, we will import the private keys【私钥】 into the wallet. To do this, execute【执行】 wallet import twice【两次】, once for each private key that was generated【生成】 earlier.

$ cleos wallet import ${private_key_1}

And then again【再次】 with the second【第二个】 private key

$ cleos wallet import ${private_key_2}

If successful【成功】, each time wallet import【导入】 command responds with the public key corresponding to your private key, your console should look something like【看起来像】..

$ cleos wallet import 5Hvgh37WMCWAc4GuyRBiFrk4SArCnSQVMhNtEQVoszhBh6RgdWr
imported private key for: EOS84jJqXj5XBz3RqLreXZCMxXRKspUadXg3AVy8eb5J2axj8cywc

We can check【检查】 which keys are loaded by calling【调用】 wallet keys

$ cleos wallet keys
[[
    "EOS6....",
    "5KQwr..."
  ],
  [
    "EOS3....",
    "5Ks0e..."
  ]
]

the wallet file itself is encrypted【加密】, so the wallet will protect【保护】 these keys when it’s locked. Accessing the keys in a locked【加锁】 wallet requires the master password that was provided to you during wallet creation【创建】.

Backing up【备份】 your wallet

Now that your wallet contains【包含】 keys, it’s good to get into the habit【习惯】 of backing up the wallet, e.g., to a flash drive or other media, to protect against the loss【损失】 of the wallet file. Without the password, the wallet file is encrypted【加密】 with high entropy, and the keys inside are incredibly difficult【苦难】 (improbable by all reasonable measures) to access.

You can find your wallet files【文件】 in the data-dir. If you did not specify【指定】 a –data-dir parameter when launching eos, your wallet files are stored【存储】 in the ~/eosio-wallet folder.

$ cd ~/eosio-wallet && ls
blockchain   blocks   config.ini   default.wallet   periwinkle.wallet

If you’ve been following the steps of this tutorial, you will the two【两个】 files, default.wallet and periwinkle.wallet. Save these files in a safe location.

Creating【创建】 an Account【账户】

Performing actions on the blockchain【区块链】 requires the use【使用】 of accounts. We use cleos to request nodeos to create accounts and publish【发布】 them on the blockchain. At this point in our tutorial, we need to start【启动】 nodeos. The following command will start a single【单一】 node testnet. See Creating and Launching【运行】 a Single Node Testnet for more on setting up a local【本地】 environment【环境】.

To start【启动】 nodeos, open a new terminal【终端】 window, go to the folder【目录】 that contains your nodeos executable【可执行文件】, and run【运行】 the following:

$ cd eos/build/programs/nodeos
$ nodeos -e -p eosio --plugin eosio::chain_api_plugin --plugin eosio::history_api_plugin

Now we can create【创建】 the account. Here is the structure【结构】 of the cleos create account command【要求】.

$ cleos create account ${authorizing_account} ${new_account} ${owner_key} ${active_key}

authorizing_account is the name of the account that will fund【找到】 the account creation, and subsequently the new【新】 account.

new_account is the name of the account you would like to create【创建】

owner_key is a public key【公钥】 to be assigned【分配】 to the owner authority【许可】 of the account. (See Accounts & Permissions)

active_key is a public key to be assigned【分配】 to the active authority of the account【账户】. of your account, and the second【第二个】 one will be permissioned for the active authority of your account【账户】.

In this tutorial, eosio is the authorizing account. The actions【行为】 performed on the blockchain must be signed【签名】 using the keys associated with the eosio account【账户】. The eosio account is a special【特殊】 account used to bootstrap【启动】 EOSIO nodes. The keys for this account can be found【找到】 in the nodeos config file, located【位于】 at ~/.local/shared/eosio/config/config.ini on Linux platforms, and ~/Libraries/Application Support/eosio/nodeos/config/config.ini on MacOS.

We need a name【命名】 for our new account【账户】. Account names must conform【确认】 to the following guidelines:

Must be less【少于】 than 13 characters

Can only contain the following symbols: .12345abcdefghijklmnopqrstuvwxyz

We will use【使用】 the name “myaccount” for the new account.

We will use the public【公开】 keys that you generated【生成】 above and imported【导入】 into your wallet (recall that the public keys begin with EOS). The keys are arbitrary【任意】 until you have assigned them to an authority. However, once【一旦】 assigned【分配】, it is important to remember【记住】 the assignments. Your owner key equates【等效于】 to full control of your account, whereas your active key equates to full access over funds【找到】 in your account.

Use cleos create【创建】 account to create your account:

$ cleos --wallet-port=8899 create account eosio myaccount ${public_key_1} ${public_key_2}

If successful【成功】, you will see output【输出】 similar to the following.

executed transaction: 7f1c6b87cd6573365a7bb3c6aa12f8162c3373d57d148f63f2a2d3373ad2fd54  352 bytes  102400 cycles
#         eosio <= eosio::newaccount            {"creator":"eosio","name":"myaccount","owner":{"threshold":1,"keys":[{"key":"EOS5kkAs8HZ88m2N7iZWy4J...
