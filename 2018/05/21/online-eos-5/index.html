Local局部 Environment环境

作者： Lian

日期： 2018年05月21日 16:05

Overview【简介】

EOSIO comes with a number of programs【程序】. The primary ones that you will use, and the ones that are covered here, are:

  • nodeos - server-side【服务器端】 blockchain node component

  • cleos - command line interface【命令行接口】 to interact with the blockchain【区块】 and to manage wallets【钱包】

  • keosd - component that manages【管理】 EOSIO wallets【钱包】

The basic relationship【关系】 between these components is illustrated in the following diagram【框图】. In the sections【节点】 that follow, you will build【创建】 the EOSIO components【组件】, and deploy【发布】 them in a single【单一】 host, single node test network (testnet) configuration【配置】.

Getting【获取】 the Code【代码】

To download【下载】 all of the code, clone the eos repository【仓库】 and its submodules【子模块】.

git clone https://github.com/EOSIO/eos --recursive

If a repository is cloned without the –recursive flag【标签】, the submodules can be retrieved【恢复】 after the fact by running this command from within【内部】 the repo:

git submodule update --init --recursive

Throughout【通过】 the EOSIO documentation【文档】 and tutorials【教程】, reference【参考】 will be made to the top level of your local【本地】 EOSIO source repository【仓库】. The location where you just cloned the eos repository is that location. The notation【记号】 ${EOSIO_SOURCE} represents【表示】 the same thing.

For example, if you ran the git clone operation【操作】 in a folder called ~/myprojects, then ${EOSIO_SOURCE}=~/myprojects/eos.

Note that ${EOSIO_SOURCE} is used in the documentation【文档】 for notational【计数】 purposes only.

No environment variable【环境变量】 is implied【暗示】 or required. More commonly, for simplicity, this and other documents might simply refer to eos. This is equivalent【等效】 to ${EOSIO_SOURCE}.

Building【创建】 EOSIO

The simple approach【方法】 to building EOSIO is to use the automated【自动】 build script【脚本】. The automated approach is documented next. Instructions【指导】 to build EOSIO manually【手动】 are described【描述】 in the section Manually【手动】 Building EOSIO, farther down in this document.

The build places content【内容】 in the eos/build folder【目录】. The executables【可执行文件】 can be found in subfolders【子目录】 within the eos/build/programs folder.

Automated【自动】 build script

There is an automated build script that can install【安装】 all dependencies【依赖】 and build【构建】 EOSIO. The script supports the following operating systems【操作系统】. We are working on supporting other Linux/Unix distributions in future【未来】 releases.

  • Amazon 2017.09 and higher.
  • Centos 7.
  • Fedora 25 and higher (Fedora 27 recommended).
  • Mint 18.
  • Ubuntu 16.04 (Ubuntu 16.10 recommended).
  • MacOS Darwin 10.12 and higher (MacOS 10.13.x recommended).

System Requirements【要求】 (all platforms)

8GB RAM free required
20GB Disk free required

Run【运行】 the build script

Run the build script from the eos folder【目录】.

cd eos
./eosio_build.sh

Build validation【验证】

Optionally【可选】, a set of tests【测试集】 can be run against your build to perform some basic【基本】 validation【校验】. To run the test suite after building, start【启动】 mongod and the run【运行】 make test.

On Linux platforms【平台】:

~/opt/mongodb/bin/mongod -f ~/opt/mongodb/mongod.conf &

On MacOS:

/usr/local/bin/mongod -f /usr/local/etc/mongod.conf &

Followed【后续】 by this on all【所有】 platforms:

cd build
make test

Install【安装】 the executables【可执行文件】

For ease【减轻】 of contract【合约】 development【开发】, content can be installed【安装】 in the /usr/local folder【目录】 using the make install target【目标】. This step is run from the build【构建】 folder. Adequate permission is required to install, so we use the sudo command【超级指令】 with make install.

cd build
sudo make install

Creating【创建】 and Launching【运行】 a Single Node Testnet【测试节点】

After successfully building【创建】 the project, the nodeos binary【二进制】 should be present in the build/programs/nodeos folder【目录】. nodeos can be run directly【直接运行】 from the build folder using programs/nodeos/nodeos, or you can cd build/programs/nodeos to change【改变】 into the folder and run the nodeos command from there.

Here, we run the command within the build/programs/nodeos folder.

You can start【启动】 your own single-node blockchain【区块链】 with this single command:

cd build/programs/nodeos
./nodeos -e -p eosio --plugin eosio::chain_api_plugin --plugin eosio::history_api_plugin

Now, keosd is automatically【自动】 started and wallet management is done by it. Other alternatives【可选】 are discussed in the Comprehensive【综合的】 Accounts & Wallets Tutorial tutorial.

Docker【容器】

Simple and fast setup【安装】 of EOSIO on Docker is also available【可行】. You can find up to date【日期】 information about EOSIO Docker in the Docker Readme.

Install Dependencies【依赖】

Docker Docker 17.05 or higher is required

Build EOSIO image【镜像】

$ git clone https://github.com/EOSIO/eos.git --recursive
$ cd eos/Docker
$ docker build . -t eosio/eos

Start nodeos docker container【容器】 only

$ docker run --name nodeos -p 8888:8888 -p 9876:9876 -t eosio/eos nodeosd.sh arg1 arg2

By default, all data is persisted【存储】 in a docker volume. It can be deleted【删除】 if the data is outdated【过期】 or corrupted【奔溃】:

$ docker inspect --format '{{ range .Mounts }}{{ .Name }} {{ end }}' nodeos
fdc265730a4f697346fa8b078c176e315b959e79365fc9cbd11f090ea0cb5cbc
$ docker volume rm fdc265730a4f697346fa8b078c176e315b959e79365fc9cbd11f090ea0cb5cbc

Alternately【可选的】, you can directly mount【锚定】 host directory into the container【容器】

$ docker run --name nodeos -v /path-to-data-dir:/opt/eos/bin/data-dir -p 8888:8888 -p 9876:9876 -t eosio/eos nodeosd.sh arg1 arg2

Get chain【链】 info

$ curl http://127.0.0.1:8888/v1/chain/get_info

Start【启动】 both nodeos and keosd containers

$ docker-compose up

After docker-compose up, two services【服务】 named nodeos and keosd will be started. nodeos service will expose【暴露】 ports 8888 and 9876 to the host. keosd service does not expose any port to the host, it is only accessible to cleos when runing cleos is running inside the keosd container as described in “Execute cleos commands” section.

Execute cleos commands【命令】

You can run the cleos commands【命令】 via a bash alias.

$ alias cleos='docker-compose exec keosd /opt/eos/bin/cleos -H nodeos'
$ cleos get info
$ cleos get account inita

Upload【上传】 sample exchange contract

$ cleos set contract exchange contracts/exchange/exchange.wast contracts/exchange/exchange.abi

If you don’t need keosd afterwards, you can stop【停止】 the keosd service using

$ docker-compose stop keosd

Change【改变】 default configuration【配置】

You can use docker compose override【覆盖】 file to change the default configurations. For example, create an alternate config file config2.ini and a docker-compose.override.yml with the following content.

version: "2"

services:
  nodeos:
    volumes:
      - nodeos-data-volume:/opt/eos/bin/data-dir
      - ./config2.ini:/opt/eos/bin/data-dir/config.ini

Then restart【重启】 your docker containers as follows:

$ docker-compose down
$ docker-compose up

Clear【清理】 data-dir

The data volume created by docker-compose can be deleted【删除】 as follows:

$ docker volume rm docker_nodeos-data-volume

Troubleshooting【故障】 Guide

1、You get an error such as St9exception: content of memory does not match data expected by executable when trying to start【启动】 nodeos

Try restarting nodeos with –resync

2、You get an error such ascould not grow database file to requested size.

Start【启动】 nodeos with –shared-memory-size-mb 1024.

A 1 GB shared memory file allows approximately【大约】 half a million transactions【交易】.

3、How do I find which version【版本】 of nodeos I’m running or connecting to?

Use cleos -H ${nodeos_host} -p ${nodeos_port} get info and you will see the version【版本】 number in the field called server_version
