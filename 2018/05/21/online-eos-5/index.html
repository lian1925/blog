Local局部 Environment环境

作者： Lian

日期： 2018年05月21日 16:05

Overview【简介】

EOSIO comes with a number of programs. The primary ones that you will use, and the ones that are covered here, are:

  • nodeos - server-side【服务器端】 blockchain node component

  • cleos - command line interface【命令行接口】 to interact with the blockchain and to manage wallets

  • keosd - component that manages【管理】 EOSIO wallets【钱包】

The basic relationship between these components is illustrated in the following diagram【框图】. In the sections that follow, you will build【创建】 the EOSIO components【组件】, and deploy them in a single host, single node test network (testnet) configuration【配置】.

Getting【获取】 the Code【代码】

To download all of the code, clone the eos repository and its submodules【子模块】.

git clone https://github.com/EOSIO/eos --recursive

If a repository is cloned without the –recursive flag【标签】, the submodules can be retrieved【恢复】 after the fact by running this command from within the repo:

git submodule update --init --recursive

Throughout the EOSIO documentation【文档】 and tutorials【教程】, reference will be made to the top level of your local【本地】 EOSIO source repository【仓库】. The location where you just cloned the eos repository is that location. The notation【记号】 ${EOSIO_SOURCE} represents【表示】 the same thing.

For example, if you ran the git clone operation in a folder called ~/myprojects, then ${EOSIO_SOURCE}=~/myprojects/eos.

Note that ${EOSIO_SOURCE} is used in the documentation for notational purposes only.

No environment variable is implied【暗示】 or required. More commonly, for simplicity, this and other documents might simply refer to eos. This is equivalent to ${EOSIO_SOURCE}.

Building【创建】 EOSIO

The simple approach to building EOSIO is to use the automated build script. The automated approach is documented next. Instructions to build EOSIO manually are described in the section Manually Building EOSIO, farther down in this document.

The build places content in the eos/build folder. The executables can be found in subfolders within the eos/build/programs folder.
